НАУЧНО-ТЕХНИЧЕСКИЙ ОТЧЕТТема: Разработка высокопроизводительного C++ DSP-движка для семантического анализа аудио и автоматического сведения музыкальных композиций.Проект: PeerifyCore Audio Engine1. АннотацияВ данном документе описывается архитектура гибридного программного комплекса, предназначенного для автоматического диджеинга (Auto-DJ). Целью разработки являлся перенос ресурсоемких алгоритмов извлечения аудиопризнаков (MIR — Music Information Retrieval), типичных для экосистемы Python (librosa), в нативную высокопроизводительную среду на базе C++ (C++17, miniaudio). Разработанная модель позволяет выполнять спектральный анализ, детектирование темпа (BPM) и поиск семантически близких композиций в реальном времени с нулевым выделением памяти (Zero-Allocation) на критических участках.2. Архитектура системыПроект реализован в парадигме микросервисной архитектуры:Frontend-уровень: Платформа Electron (HTML/JS), обеспечивающая графический интерфейс и управление потоком выполнения.Backend-уровень (Ядро): Динамически подключаемая библиотека (DLL) на С++, взаимодействующая с фронтендом через Foreign Function Interface (FFI) с использованием библиотеки koffi.Хранилище данных: Легковесная NoSQL-база данных на основе формата JSON (nlohmann::json), хранящая предварительно вычисленные акустические слепки (Fingerprints) композиций.3. Алгоритмы цифровой обработки сигналов (DSP)Для получения акустического слепка композиции применяется пайплайн, состоящий из оконного анализа во временной и частотной областях.3.1. Подготовка сигнала и ПреэквализацияАудиосигнал конвертируется в монофонический формат с фиксированной частотой дискретизации. Для компенсации спектрального спада (естественного преобладания низких частот) применяется цифровой фильтр высоких частот (Pre-emphasis):$$y[n] = x[n] - 0.97 \cdot x[n-1]$$Это позволяет выровнять амплитудно-частотную характеристику и повысить точность извлечения спектральных признаков.3.2. Оконное сглаживание и БПФСигнал сегментируется на перекрывающиеся фреймы (размер окна $N = 4096$, шаг перекрытия $N/2$). Для подавления эффекта растекания спектра (Spectral Leakage) к каждому фрейму применяется оконная функция Хэннинга:$$w[n] = 0.5 \cdot \left(1 - \cos\left(\frac{2\pi n}{N-1}\right)\right)$$Трансформация во временную область осуществляется с помощью итеративного алгоритма Быстрого преобразования Фурье (Radix-2 Cooley-Tukey FFT) с битовой реверсией. Алгоритмическая сложность составляет $O(N \log N)$, при этом алгоритм работает in-place, минимизируя нагрузку на кэш процессора.4. Экстракция акустических признаков (Feature Extraction)Система генерирует 24-мерный вектор (Fingerprint), состоящий из двух логических блоков: тембрального и гармонического.4.1. Тембральная статистика (12D)Из каждого фрейма извлекаются 6 базовых характеристик:Log-RMS: Логарифмическая среднеквадратичная амплитуда (динамика).Zero-Crossing Rate (ZCR): Частота пересечения нуля (шумность/перкуссивность).Spectral Centroid: Центр масс спектра, определяющий "яркость" тембра.Spectral Rolloff: Частота, ниже которой сосредоточено 85% спектральной энергии.Spectral Bandwidth: Эффективная ширина полосы пропускания.Spectral Flatness: Мера тональности сигнала (отношение среднего геометрического спектра к среднему арифметическому).Для обеспечения масштабной инвариантности частотные признаки нормализуются по частоте Найквиста ($f_s / 2$). Итоговый 12D суб-вектор формируется путем вычисления математического ожидания (Mean) и стандартного отклонения (StdDev) для каждой из 6 характеристик по всем фреймам.4.2. Хрома-профиль (12D Chroma Features)Для оценки гармонической структуры (тональности) спектр сворачивается в 12-мерный вектор классов высоты тона (от $C$ до $B$). Частота каждого бина $f$ соотносится с музыкальной шкалой по формуле:$$Pitch = 12 \cdot \log_2\left(\frac{f}{C_0}\right) \pmod{12}$$где $C_0 \approx 16.35$ Гц. Полученные гистограммы усредняются по всему треку.4.3. $L_2$-НормализацияДля предотвращения доминирования признаков с большой дисперсией, финальный 24D вектор $\mathbf{v}$ подвергается $L_2$-нормализации:$$\mathbf{v}_{norm} = \frac{\mathbf{v}}{\sqrt{\sum_{i=1}^{24} v_i^2}}$$5. Анализ ритмической структуры (BPM)Детектирование темпа производится независимым алгоритмом:Вычисление огибающей энергии (Energy Envelope) с размером блока 1024 семпла.Вычисление функции активации (Onset Detection) путем дифференцирования огибающей.Нормализованная автокорреляция функции активации для поиска доминирующего лага (периодичности).Коррекция октавных ошибок (Octave Error Correction): принудительное масштабирование результата в рабочий диджейский диапазон $70 \le BPM \le 180$.6. Метрика семантической близостиДля поиска релевантных композиций используется косинусное сходство (Cosine Similarity) между акустическими слепками $\mathbf{A}$ и $\mathbf{B}$:$$S_C(\mathbf{A}, \mathbf{B}) = \frac{\sum_{i=1}^{n} A_i B_i}{\sqrt{\sum_{i=1}^{n} A_i^2} \sqrt{\sum_{i=1}^{n} B_i^2}}$$Для формирования итогового рейтинга схожести (Strict Score) и подавления ложноположительных срабатываний применяется нелинейное масштабирование с кубическим штрафом:$$Score = \max(0, S_C)^3$$7. ЗаключениеПеренос DSP-алгоритмов из Python в C++ позволил сократить время сканирования и извлечения признаков на порядки, избавив конечный продукт от тяжеловесных зависимостей. Математический аппарат обеспечивает высокую точность кластеризации жанров и автоматического подбора треков для Auto-DJ, приближаясь по качеству к проприетарным решениям на рынке музыкального программного обеспечения.
